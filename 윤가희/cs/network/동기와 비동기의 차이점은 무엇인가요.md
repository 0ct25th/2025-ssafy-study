## 내 답변
```
동기는 작업을 순서대로 처리하며, 한 작업이 완료될 때까지 다음 작업이 시작되지 않고 대기합니다.
비동기는 작업을 동시에 처리하며, 한 작업이 완료될 때까지 기다리지 않고 다음 작업을 바로 시작합니다.
```

<br>

## 내 정리
- **동기(Synchronous)**
    - 작업을 **순서**대로 처리
    - 한 작업이 완료될 때까지 다음 작업은 시작되지 않고 기다림
    - **블로킹(Blocking) 방식**
- **비동기(Asynchronous)**
    - 작업을 **동시**에 처리하는 모델
    - 한 작업의 시작을 요청한 후, 그 작업이 완료될 때까지 기다리지 않고 다음 작업을 바로 시작
    - **논 블로킹(Non-Blocking) 방식**
- **블로킹(Blocking)**
    - 함수를 호출하면 **호출된 함수가 작업을 마칠 때까지 제어권을 가짐**
    - 호출한 함수는 제어권을 돌려받을 때까지 대기
- **논 블로킹(Non-Blocking)**
    - 함수를 호출하면 호출된 함수가 **제어권을 즉시 호출한 함수에게 반환**
    - 호출한 함수는 대기하지 않고 바로 다음 작업 수행
- **블로킹/논블로킹과 동기/비동기의 관계**
    - 서로 조합
    - 동기/비동기가 **전체 작업 흐름**을 결정
    - 블로킹/논블로킹은 **함수 간 제어권**을 어떻게 처리할지 결정
    <br>
    
    | | 동기 | 비동기 |
    |:---:|:---:|:---:|
    | 블로킹 | **가장 일반적인 형태** (호출된 함수가 작업을 마치고, 결과를 반환할 때까지 대기) | **비효율적이고 사용되지 않음** (블로킹이 발생하지만, 결과는 콜백으로 받음) |
    | 논 블로킹 | **거의 사용되지 않음** (결과를 기다리지 않고 다음 작업을 수행하지만, 결과 확인을 위한 로직을 직접 구현해야 함) | **이상적인 형태** (호출 즉시 제어권을 반환하고, 작업이 완료되면 알림을 받음) |
